# BOSS Helper Manual Test Checklist

## 安装检查

- [ ] Tampermonkey 已启用。
- [ ] `boss-helper.user.js` 已保存。
- [ ] 脚本匹配 `https://www.zhipin.com/*`。
- [ ] 打开 BOSS 职位列表页后，右下角出现 BOSS Helper 面板。

## 默认配置检查

- [ ] 间隔秒数默认是 `60`。
- [ ] 每日上限默认是 `150`。
- [ ] 面板显示成功、跳过、失败、剩余额度。

## 单次成功流程

- [ ] 手动登录 BOSS。
- [ ] 手动设置职位筛选条件。
- [ ] 将每日上限改成 `1`。
- [ ] 点击 `开始`。
- [ ] 脚本点开左侧职位。
- [ ] 脚本点击右侧 `立即沟通`。
- [ ] 页面出现 `已向BOSS发送消息` 弹窗。
- [ ] 脚本点击 `留在此页`。
- [ ] 成功计数变成 `1`。
- [ ] 脚本因为达到每日上限而停止。

## 跳过流程

- [ ] 找到一个按钮为 `继续沟通` 的职位。
- [ ] 点击 `开始`。
- [ ] 脚本跳过该职位。
- [ ] 成功计数不增加。
- [ ] 跳过计数增加。

## 停止流程

- [ ] 页面出现登录、验证码或安全验证时点击 `开始`。
- [ ] 脚本立即停止。
- [ ] 面板日志显示停止原因。

## 列表滚动

- [ ] 当前可见职位处理完成后，脚本向下滚动左侧职位列表。
- [ ] 新职位加载后继续处理。
- [ ] 列表到底时，脚本停止并提示没有更多职位。

## 回归检查

- [ ] 脚本没有点击 `继续沟通`。
- [ ] 脚本没有进入聊天页。
- [ ] 脚本没有发起额外网络请求。
- [ ] 脚本没有保存招聘者隐私信息。
